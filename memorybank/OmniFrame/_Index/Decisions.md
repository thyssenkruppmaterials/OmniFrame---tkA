- [[ADR-SQCDP-Measured-vs-Viewport-Hero-Typography]] — 2026-05-17 — v15.2 ADR, **supersedes [[ADR-SQCDP-Uniform-vs-AutoFit-Typography]] (v15.1)**. Third same-day decision on SQCDP TV hero typography. The user's v15.1 TV screenshot showed every primary card overflowing or wrapping (Delivery `99.7%` clipped right-edge; Safety `848 Days` + Quality `35 QNs` wrapped; Shipping `500 Orders` + Announcement `73 DAYS FOR PHYSICAL` rendered at the primary clamp and overflowed worse because v15.1's gate didn't split tiers). Pure-CSS approaches stop here: CSS units can't deliver "every card the same px AND no card overflows" without measuring what actually fits. The v15.1 ADR's "trigger to revisit" ("a curator authors a value that's longer than what comfortably wraps at the vh size") fired on the canonical 5-card layout itself. **Decision:** JS measurement via `useUniformHeroFit` + a context provider, with three independent tier registries (`primary` / `sub` / `secondarySingle`). Each tier picks `min(maxFit across tier)` clamped by `viewportHeight * viewportCeilingVh[tier] / 100`, floored at `floorPx[tier]`, rounded to 4 px. Overflow fallback demotes the snuggest entry to 2-line wrap when `min(maxFit) < floor`, then picks tier size from survivors. Curator override defers via `enabled: false`. Reversal of the v15.1 decision to reject JS measurement — the user's screenshot is the proof that the user's preference for fit-and-uniform outranks the v15.1 concerns ("adds JS", "sibling coupling", "first-paint flash") which v15.2 addresses with debounced/RAF-collapsed compute, refs-of-everything stable ref callback, and `opacity-0` first frame. Companion implementation [[Implement-SQCDP-Measured-Hero-Typography]]; pattern extension in [[Patterns/TV-Viewport-Fit-Grid]].
