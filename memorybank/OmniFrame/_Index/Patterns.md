- [[Job-Queue-Drain-Mode]] — Claim-back-to-back drain pattern for a push-notification + fallback-poll job queue (v1.7.0). Eliminates the 30-60s inter-job dwell penalty that fixed-interval polling produces when Realtime push events are missed. Bounded via a burst cap (`_DRAIN_MAX_CHAIN = 50`) for shutdown responsiveness. Forms a defensive trio with [[Self-Healing-Schema-Fallback]] (PostgREST schema cache layer) and [[Debug/Fix-Missed-Realtime-Events-Backfill]] (upstream Realtime event layer) — each layer's primary fast path has a cheap, bounded, quiet recovery mechanism. - [[Async-Library-Circuit-Breaker]] — Catch → count → trip → fall back → auto-recover containment for asyncio-based third-party libraries that crash and flood stderr at high frequency (v1.7.1). First applied to contain the `realtime>=2.x` `_reconnect()` bug (`asyncio.wait([])` → `ValueError`) which wedged the OmniFrame Agent v1.7.0 after ~25min idle on a Citrix VDA. Combines a custom `loop.set_exception_handler(...)` (Layer A), a sliding-window deque-backed counter (Layer B), an idle-backoff fallback path that tightens its polling when the breaker trips (Layer C), and a 5min auto-recovery daemon (Layer D'). Pairs with [[Self-Healing-Schema-Fallback]] (per-column 5min cooldown one layer down) and [[Job-Queue-Drain-Mode]] + [[Debug/Fix-Missed-Realtime-Events-Backfill]] (the trio: drain mode + backfill + circuit breaker = three independent recovery layers, each with a cheap, bounded, quiet recovery mechanism). See [[Debug/Fix-Realtime-Library-CrashLoop]] for the original incident.

- [[TV-Viewport-Fit-Grid]] — 2026-05-17 — Three coordinated changes that make a multi-tier TV board fit a single viewport snapshot with no scroll and no internal dead space: (1) board grid wrapper becomes `flex h-full flex-col` in TV mode, (2) tier rows share the vertical budget via `flex-N min-h-0 auto-rows-fr`, (3) card bodies drop the hard `min-h-[…]` value reserves and use `flex-1 items-end` against a flex-1 wrapper instead — baseline alignment now comes from the row's stretched height, not a hardcoded 16rem floor. First applied to SQCDP — see [[Implement-SQCDP-TV-Viewport-Fit]]. **Extended 2026-05-17 with a Dynamic counts variant** (see [[Implement-SQCDP-Editable-Categories]]) that retires the hard-coded 5+3 split for a `resolveGridSizing(primaryCount, secondaryCount)` helper in `lib/grid-sizing.ts` — primaries get a 1.5× multiplier on the secondary tier's count (chart strip earns it), clamped at 6 cols / flex-8, with empty tiers hiding the row entirely. **Further extended 2026-05-17 with an Auto-fit hero typography section** (see [[Implement-SQCDP-Hero-Autofit-Typography]]): a 5-step recipe layering CSS container queries + `cqh` units on top of the viewport-fit chain so the hero number scales with the card's actual rendered height. **The cqh recipe was SUPERSEDED same day by a v15.1 Uniform fluid typography subsection** ([[Implement-SQCDP-Uniform-Hero-Typography]] + [[ADR-SQCDP-Uniform-vs-AutoFit-Typography]]) — `cqh` resolves against each card's own value-block height which produces visibly inconsistent glyph sizes across siblings; `vh` resolves against the viewport so every card on a given screen renders at the same px while still scaling smoothly across 1080p / 1440p / 4K. The v15 recipe is preserved as historical; new work should follow v15.1. Pairs with [[Elevated-KPI-Stat-Cards]] (the unchanged card shell) and [[Cinematic-Tab-Rotation]] (the alternative when content genuinely can't fit even at full viewport).

- [[Hold-To-Confirm-Gesture]] — 2026-05-17 — Press-and-hold (~800ms, same constant as the kit-picking form's `autoAdvanceDelay`) confirmation gesture for UI actions that have no other verification gate (no barcode, no server-side cross-check) and where a single accidental tap would be a meaningful operator-side error.
