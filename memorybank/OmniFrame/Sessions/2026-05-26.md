# 2026-05-26

## AgentMaster crash on Citrix non-persistent VDIs — "Could not create temporary directory!"

Field report: `OmniFrame_AgentMaster.exe` fails to launch on a customer's Citrix non-persistent build with the bootloader-level dialog *"Could not create temporary directory!"*; `OmniFrame_Agent.exe` exits silently with the same root cause. The customer's `%TEMP%` is redirected/locked by App Layering + Profile Management and gets wiped mid-session, so the PyInstaller onefile `_MEIxxxxxx` extraction has nowhere writable to land.

Decision: pin the bootloader's extraction folder to a `_omniframe_tmp\` sibling of the .exe via launcher `.bat` shims rather than hard-coding `--runtime-tmpdir` at build time. The launcher approach preserves the "launch the .exe directly works on healthy machines" property and inherits cleanly into subprocesses spawned by the supervisor.

### Built
- `omni_agent/launchers/OmniFrame_AgentMaster.bat`, `.../OmniFrame_Agent.bat`, `.../OmniFrame_Connect.bat` — each launcher sets `TEMP`/`TMP` to `%~dp0_omniframe_tmp` then runs the sibling `.exe` (windowed GUIs via `start ""`; the console worker inline so its cmd window doubles as a live log view).
- `omni_agent/build_exe.bat` — added `robocopy` of `launchers/`, a staging step that copies the three .bat files into `dist/`, included the matching launchers in both `Compress-Archive` calls, rewrote `dist\README.txt` to a four-line operator-facing explanation, and refreshed the end-of-build summary to mention the Citrix entry point.
- `omni_agent/master/build/README.md` — new "Citrix non-persistent launcher .bat" section + dist-layout table now lists the .bat artifacts.

### Tested
- `omni_agent/master/tests/test_citrix_launchers_static.py` — new, 13 cases. Locks in launcher existence, the `TEMP`/`TMP` pinning pattern, the windowed-vs-console `start ""` contract (with REM-line filtering so the banner prose doesn't false-positive), and the build_exe.bat wiring (robocopy + dist copy + zip inclusion + no `--runtime-tmpdir` regression guard).
- `python3 -m pytest omni_agent/master/tests/ -q` — 215 passed, 0 failed. Existing `test_phase_g_packaging_static.py` (6 cases) still green; the worker/master PyInstaller invocations are byte-for-byte unchanged.
