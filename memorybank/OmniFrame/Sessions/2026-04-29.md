### Decisions made worth user review
1. **Polling always runs alongside hooks.** Even when hooks engage successfully, polling provides a safety net for sbar/screen events the hooks don't surface and protects against silent hooks failure. Slight CPU cost; large reliability win.
2. **Recording deletion is hard-delete.** No trash/recycle. Spec said "hard-delete (file unlinked)" — implemented exactly that.
3. **Replay safety gate is explicit-opt-in via header.** `X-Recording-Allow-Replay: yes`. The UI only sends it after a confirm modal. A bare `curl -X POST /sap/recording/{id}/replay` from outside our UI returns 'opt-in required'.
4. **Encryption is fail-soft.** If `cryptography` isn't bundled into the EXE, we write plaintext with a console warning rather than refusing to record. Acceptable because the file lives only in `%LOCALAPPDATA%`.
5. **Recordings are not registered into `QUERY_LIBRARY` automatically.** The translator outputs Python that the user pastes into `agent.py` + rebuilds the EXE. Doing it dynamically would require a hot-reload mechanism that's incompatible with PyInstaller's frozen bundle.

### Decisions worth user review
1. **Bulk export is now the default** instead of an opt-in heuristic (LT10 used `storage_type == '*'` to opt in). LT22 result sets are large enough that the bulk export should always win — the `use_bulk_export` field on the request gives us a remote kill-switch without needing an agent rebuild.
2. **COM errors re-raise inside the extractors** instead of being absorbed by the existing `try/except Exception:` fallback. A COM crash means the SAP bridge is dead — the fallback would just trigger the same crash. Letting it bubble lets the outer handler do the right thing (flip `sap_connected`).
3. **`_auto_select_valid_session()` pre-flight check** runs before patching the run row to "running". Cleaner UX (the user sees "failed: SAP not available" instead of "running" → "failed: traceback") and prevents the pre-1.6.3 silent-failure loop where the agent kept claiming jobs against a dead bridge.
4. **SmartImportButton "upgrade hint" path** trades a tiny visual nag for clear discoverability. Pre-fix, an old EXE meant the agent option silently vanished; the user had no way to know "Import via Agent" was a thing the agent could do if rebuilt. Post-fix, they see the disabled option + the version number + the upgrade hint.
5. **Capabilities catch-up** is just hygiene — `supabase-session`, `agent-supabase-logout`, and `truststore-tls` are all already-shipping endpoints/features that weren't surfaced via `/health.capabilities`. Surfacing them now lets the frontend's `hasCapability(...)` gates light up correctly going forward.
