## Cancel Kit TO Line

Added per-line cancellation to the Kit Build Audit Trail so operators can mark a TO line as cancelled with a captured reason. The kit keeps advancing on the remaining live lines and the cancelled row stays visible for audit history.

- **Migration 325** — `ALTER TABLE RR_Kitting_DATA ADD COLUMN cancelled BOOLEAN NOT NULL DEFAULT FALSE`, `cancelled_at TIMESTAMPTZ`, `cancelled_by_user UUID REFERENCES user_profiles(id)`, `cancelled_reason TEXT`. CHECK constraint `rr_kitting_data_cancellation_invariants` enforces all-or-nothing population + non-empty reason. Partial index on `(kit_serial_number) WHERE cancelled = FALSE` accelerates the predominant active-line reads. Applied to `wncpqxwmbxjgxvrpcake`; columns + constraint + index verified.
- **Service plumbing** — `RRKittingDataService` gets a new `cancelTOLine(toLineId, reason)` method (validates non-empty trim + authenticated, casts the UPDATE through `unknown` until `database.types.ts` is regenerated). Both `getKitBuildPlanDetailsBy*` methods now surface cancellation fields on `TOLine` and use `activeToLines = toLines.filter(t => !t.cancelled)` for stage gating. `recheckBomCoverageBySerial` filters cancelled rows from the `toMaterials` set so cancelling the only TO for a required material correctly leaves it uncovered. `KitKanbanService.computeKitProgress` filters cancelled rows from totals so the kanban card doesn't get stuck.
- **CancelTOLineDialog** (NEW) — modal that captures the reason in a `<Textarea>` (max 500 chars), reseeds on every open, Confirm disabled until non-empty trim. Header surfaces TO #, material, description for unambiguous targeting.
- **Kit Build Audit Trail integration** — new rightmost actions column in the TO Lines table with a per-row Trash2 ghost button. Cancelled rows render with `bg-muted/40 text-muted-foreground` + line-through on the data cells, em-dashed Picked/Kitted, and a destructive-tint `Cancelled` pill on the Material cell with `title="reason · actor · timestamp"`. New neutral `Cancelled` count pill in the table header. Pending / Complete pills now reflect *active* workload only.
- **Audit trail integration** — every cancellation stamps a `kit_notes` system note (`event_kind = 'to_line_cancelled'`) via the existing `addSystemNote` hook — same surface used by flag add/remove and ship-short authorize events. Reads `TO {N} / Material {M} cancelled. Reason: {reason}`.

## Debug — Black Hat invisible inside Kit Build Audit Trail for KIT-20260518-005

User reported that `KIT-20260518-005` was showing a Black Hat on the kanban card but the audit-trail dialog had no recovery affordance (no Black Hat pill in the Build Flags bar, no `BlackHatShipShortPanel`, no `Add TO to Clear Black Hat` button).

Root cause: a single legacy `kit_build_flags` row with `kit_serial_number IS NULL`, set on 2026-05-11 — pre-dating the [[Kit-Serial-Scoping]] / migration 303 per-serial scoping rollout. The dialog's reader (`getActiveFlagsBySerialNumber`) is strictly `.eq('kit_serial_number', …)` so the NULL-serial flag was invisible to it. The kanban card's flag chip uses a more permissive PO-keyed path which is why the chip *was* visible there.

DB scan confirmed exactly **one** such row in the entire production DB — not a systemic issue. Applied a targeted one-row UPDATE backfilling `kit_serial_number = 'KIT-20260518-005'`. Post-fix scan returned 0 remaining NULL-serial active flags.

User opted for the smallest-possible-change path (option A) over a defensive service hardening (option B) since (a) only one row was ever affected, and (b) a PO-fallback would re-introduce the cross-kit-flag-bleed bug that originally motivated the per-serial scoping.

After-fix UX: the BlackHatShipShortPanel now renders inside the dialog with the three missing BOM components (`M250-10296`, `23086020`, `M250-10858`) and exposes both recovery on-ramps — [[Add-TO-To-Clear-Black-Hat]] for re-covering the materials, or [[Edit-Ship-Short-Post-Creation-Flow]] / [[Authorized-Ship-Short-Negates-Black-Hat]] for authorising them as ship-short.

Captured as [[Fix-Legacy-Null-Serial-Black-Hat-Hidden-From-Dialog-2026-05-21]].
