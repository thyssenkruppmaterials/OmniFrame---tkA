### Defect

`public.sync_cycle_count_to_work_task()` UPDATE branch contains:

```sql
priority = COALESCE(NEW.priority, priority),
```

`NEW.priority` is `cycle_count_priority` (enum on `rr_cyclecount_data`); the LHS column `priority` is `text` (on `work_tasks`). PostgreSQL has no implicit common type between an enum and `text` for `COALESCE`, so the trigger fails with `42804` on every UPDATE. Same root cause class as the migration 261 patch from earlier today (`work_engine_drift` view needed `rcc.priority::text`); the trigger function carries the same bug because it had not been exercised (`work_tasks_shadow_write` was false everywhere). The Phase 11.1 kickoff is the first action that flipped the flag, so this is the first run that surfaced it.

INSERT branch (`COALESCE(NEW.priority, 'normal')`) works — the unknown literal resolves to enum and the implicit enum→text *assignment* cast on the column accepts it. Only the UPDATE branch fails.

### Rollback

1. `UPDATE work_engine_settings SET feature_flags || '{"work_tasks_shadow_write":false}' WHERE organization_id = 'c9d89a74-7179-4033-93ea-56267cf42a17'` → `(false, false, false)` at `2026-05-02T21:27:22Z`.

### Secondary finding — backfill mechanics

Even after the priority cast is patched, the backfill script (`scripts/backfill/work_tasks_from_cycle_count.mjs`) won't actually project historical rows: it bumps `updated_at` on each chunk to fire the trigger, but the trigger's UPDATE branch only updates *existing* `work_tasks` rows. For the 8908 missing-projection rows, the UPDATE matches zero rows and is a silent no-op. Two ways forward:

1. Change the trigger UPDATE branch to an UPSERT (`INSERT … ON CONFLICT (id) DO UPDATE`).
2. Or have the backfill script issue a direct `INSERT … SELECT` mirroring the trigger's INSERT branch logic (with `app.skip_sync` GUC set to suppress the reverse trigger).

Option 1 is cleaner — it makes the trigger semantically correct under both fresh and pre-existing projection states and keeps the backfill driver's "bump updated_at" model viable.
