# Session — 2026-05-13

## SQCDP Editor Dialog v12.3 — side-by-side preview / controls layout

User reported a layout bug in the **Edit metric · {Title}** dialog: form fields (subtitle, Format, Period, Current value, Target) were visibly floating beneath the sticky live-preview card. Asked for a fix and an overall layout improvement — preview on the LEFT, controls on the RIGHT.

### Diagnosis

The v12 layout used `bg-muted/20 backdrop-blur-sm` on the sticky-top live-preview wrapper. As the curator scrolled, the form inputs scrolled UP behind the translucent backdrop and were visible through it. Frosted-glass treatment that intentionally lets background show through.

### Fix shape

Reshaped the dialog body into a **side-by-side flex split**:

- `<DialogContent>`: `flex flex-col`, `sm:max-w-[1040px]` (was 920).
- Body: `flex min-h-0 flex-1 flex-col overflow-hidden`.
- `MetricForm`'s `<form>`: `flex flex-col md:flex-row` with `md:overflow-hidden`.
  - **Left `<aside>`** (`md:w-[380px]`): `bg-muted/20`, `md:overflow-y-auto`, `md:border-r`. Hosts `<LivePreview>` only.
  - **Right `<div>`** (`flex-1 md:overflow-hidden`): `flex flex-col` with a `flex-1 md:overflow-y-auto` content area for `<Tabs>` and a `shrink-0 border-t` `<DialogFooter>` pinned at the bottom (`md:static`, `sticky bottom-0` on mobile).
- `<LivePreview>` lost the `sticky top-0 z-10` + `bg-muted/20` + `backdrop-blur-sm` chrome and the `mx-auto max-w-md` centering; the card now fills the column.
- `<ProblemForm>` got a matching scroll wrapper + sticky footer (single column, no preview).
- Mobile (`< md`): stacked single-column scroll with bottom border between preview and form.
