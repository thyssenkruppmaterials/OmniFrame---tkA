## Build sheet TO barcodes were NOT scannable — fractional width bug
User asked to verify the TO barcodes on the Kit Build Sheet scan flawlessly.
Empirical check (Playwright harness with the exact production jsbarcode
options → zxing-cpp decode) proved they **did not decode at all**: raw PNG,
28px screen render, and simulated 600dpi print all failed, while the kit QR
on the same sheet decoded fine. Root cause: `width: 1.4` — fractional module
width gets anti-aliased on canvas, destroying Code128 bar-width ratios; also
`margin: 0` baked in no quiet zone. Fixed in `kit-build-sheet.tsx`:
`width: 2`, `height: 56` (2× display height), `marginLeft/Right: 20`,
dropped `imageRendering: 'pixelated'`. All conditions now decode (numeric +
alphanumeric round-trip). ([[Fix-Build-Sheet-TO-Barcode-Not-Scannable-Fractional-Width]])
Flag: `ui/barcode.tsx` is a fake placeholder still used by putback-tool-form.
