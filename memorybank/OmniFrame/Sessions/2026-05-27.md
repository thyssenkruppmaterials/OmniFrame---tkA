# 2026-05-27

## LL01 Warehouse Activity Monitor — Aging tab (warehouse / quarter / user breakdowns)

User asked to extend the existing Plant × Category heatmap with three new dimensions, sourced from the manual `LL01_Worker_Full/LL01 Stack.xlsx` spreadsheet they pull from SAP today:
1. Per-warehouse aging buckets — items aged `>30 / >60 / >90 days`.
2. Per-quarter trend going back as far as the data spans.
3. Per-user breakdown for any category whose underlying SAP rows expose a user column.

### Audit

- Existing pipeline (worker `omni_agent/ll01_warehouse_activity_monitor.py` → HTTP/job-queue → frontend `<WarehouseActivityMonitorView />` → `HeatmapTab` + `TrendTab` + Supabase `ll01_activity_snapshots` for trend history) is healthy. Direct agent route, no realtime channel — no [[Realtime-Policy]] interaction.
- The Excel revealed the worker was dropping three columns it could trivially capture: `open_to.created_on`/`created_by`, `open_tr.created_on`, `open_posting.created_on`. Without these the frontend can't compute aging at all on the most-touched categories.
- Data span verified per-category: Open TR goes back to **2010-04-29** (16+ years); Open TO to 2016; Critical Delivery to 2020; the rest of the categories range from 2019–2024 to 2026.

### Design choice

- **New "Aging" tab** sibling to Heatmap / Trend (the brainstorm note explicitly evaluates inline-below-heatmap vs. a new tab and lands on tab — the page is dense enough that sibling tabs read clearer than scrolling).
- **Cumulative bucket semantics** (`>30` includes `>60` includes `>90`) — documented inline + via hover tooltip on each cell. Discrete bins (`30-60 = gt30 - gt60`, `60-90 = gt60 - gt90`) are derivable.
- **Frontend-only derivation** via `useMemo` on the existing `result.categories[i].rows` payload — no second fetch, no new TanStack Query, no realtime channel.

### Built

- `omni_agent/ll01_warehouse_activity_monitor.py` — additive column-spec extension for `open_to` (`Created On`, `Created by`), `open_tr` (`Created On`), `open_posting` (`Created On`). Response envelope gains `payload_version: 2` so older runs degrade gracefully.
- `src/features/admin/sap-testing/components/warehouse-activity-monitor-types.ts` — added `LL01_AGING_DATE_KEY` + `LL01_USER_FIELD` maps, `LL01AgingBuckets` shape, `bucketizeLL01Aging`, `ll01QuarterLabel`, `ll01QuarterSortKey`. Drilldown column orders also extended to surface the new fields.
- `src/features/admin/sap-testing/components/AgingTab.tsx` — new tab with three cards: Plant × Aging-bucket grid (cumulative `>30/>60/>90` + Dated / Undated), Items-per-quarter stacked bar chart (Recharts, plant-coloured), Top-users table with "Not available for this category" fallback for the four categories without a SAP user column. v1-payload runs render an upgrade hint instead of a broken UI.
- `src/features/admin/sap-testing/components/warehouse-activity-monitor-view.tsx` — wired the new tab between Heatmap and Trend.
