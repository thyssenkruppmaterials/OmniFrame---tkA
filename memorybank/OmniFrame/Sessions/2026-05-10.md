### Decisions

- **Shell-level WS handler vs per-lane**. Subscribed once at the dispatcher shell rather than mounting `useWorkerTasks(…, { enableRealtime: true })` per lane (which would stack 6 handlers for 6 lanes). See [[ADR-Dispatcher-Shell-WS-Handler]]. The dialog continues to use `useWorkerTasks` per-instance, sharing the same `WORKER_TASKS_QUERY_KEY` cache.
- **Cross-lane FLIP — kept (with fallback)**. Implemented via `<motion.div layout>` on the lane container + `<motion.li layout='position'>` on each row + `<AnimatePresence>` for source-exit / target-enter. Did NOT use a one-frame shared `layoutId={`task-${countId}`}` because the source/target lanes share the same `<DndContext>` — the layout-position interpolation produces the same visual without the brittle one-frame coordination dance. Spring enter/exit fallback is in place because the same code path drives a normal arrival and a cross-lane arrival.
- **Virtualisation thresholds**. `@tanstack/react-virtual` kicks in when per-lane tasks > **12** OR total visible across all lanes > **110**. Below either threshold, plain DOM is faster (virtualisation adds ResizeObserver + scroll-measure cost on every paint).
- **In-progress tasks**. Drag-disabled at the source via `useSortable({ disabled: true })` + visual cue (`cursor-not-allowed` grip + dim icon + ARIA-labelled "Cannot reassign in-progress task"). Defence-in-depth client-side reject in `use-cross-lane-reassign.ts` if a programmatic caller bypasses. **No auto-release-and-push** — the spec called this out as SHOULD-not-MUST and the destructive flow needs explicit consent. Captured as a follow-up.
- **Two droppables per lane** (`after-now` + `end-of-pipeline`) so the supervisor sees distinct "Drop after current" vs "Drop at end" hints during cross-lane drag.
- **RBAC**. Migration 294 seeds `tab_definitions` + `role_tab_permissions` for admin / manager / superadmin / logistics_coordinator. Migration is committed but **not auto-applied** — the user runs it via Supabase MCP when ready. Until then, only admin/superadmin see the tab via the existing role bypass in `useTabPermissions`.
- **Cross-surface localStorage sync**. The per-operator dialog and the dispatcher lane both render `useOperatorTaskQueueOrder` for the same operator id. Drag context writes localStorage directly + dispatches a synthetic `storage` event; the hook's new listener re-reads. Side benefit: multi-tab supervisors stay coherent for free (native cross-tab `storage` events use the same shape).
- **`<MotionConfig reducedMotion="user">`** at the tab boundary so every spring / transition / glow honours OS prefer-reduced-motion automatically.
- **6s ghost-lane TTL** when an operator goes offline. Preserves the supervisor's spatial memory of where each lane was; a simpler immediate-unmount caused jarring layout reflows during testing.

## Production Boards — v4 Operating-Hours Window 6 AM – 7 PM

Trimmed the Hourly Completion Tracker from a 24-column grid to the 13 hour buckets the building is actually open for: `BOARD_HOURS = [6…18]` — i.e. 6 AM through the 6 PM hour ending at 7 PM. Operating window is the half-open interval `[BOARD_OPENING_HOUR, BOARD_CLOSING_HOUR) = [6, 19)`. Constants centralised in `production-boards/lib/hour-bucket.ts`.

Key behaviours:

- `bucketEventsByHour` drops events outside `[6, 19)` so KPI strip totals + per-row `userTotal` reflect operating-hours activity only.
- `computeHoursElapsed` rebuilt around the new origin: 13h historically; clamp `(now − 6 AM)/60` to `[5/60, 13]` during the day; **0** before 6 AM (pre-open); **13** after 7 PM. ### Bug 2 — `localhost:8765/health ERR_CONNECTION_REFUSED` despite the suppressLocalProbe mechanism

**Root cause** — effect ordering. `useExecutionMode()` calls `useAgentDetection()` first thing in its body; React runs effects in declaration order, so `useAgentDetection`'s effect (which calls `startPoller() → probeOnce()` → fires `/health` fetch immediately) runs BEFORE the `inventory-management-tab.tsx` `useEffect` that registers `suppressLocalProbe(token)`. Module load runs BEFORE any React render, so by the time `useAgentDetection`'s `startPoller()` fires its first probe, suppression is already active and the probe short-circuits via `isLocalProbeSuppressed()` without making the network call. ## Production Boards — v7 Edit-toggle bug fix (cross-component URL-state sync)

## TKAJSM parser undercount fix (LX25 Inventory Completion)

User reported the **TKAJSM** variant of LX25 Inventory Completion returned **17 bins** for warehouse JSM (actual ~1,400-1,500). Other variants (TKAWH5/WH8/JSF/PDC) returned correct totals.

### Root cause

`_STORAGE_TYPE_RE = re.compile(r"^(\d{1,4})\s+(.+?)\s*$")` in `omni_agent/lx25_inventory_completion.py` was **digits-only** — JSM uses **alpha** storage type codes (`DDN`/`DDS`/`DDU`/`DUR`/`SCD`), so none of JSM's 5 storage-type marker lines matched. Every metric row fell through to the `current_st is None` defensive branch and wrote into a single shared `(unspecified)` placeholder; each new `Total number of bins` overwrote the previous, leaving the warehouse total = the LAST storage type's count (SCD = 17). Other warehouses use numeric codes (`110`, etc.) so they parsed cleanly.

### Fix

Broadened the regex to `^([A-Z0-9]{1,4})\s+(.+?)\s*$` — uppercase alphanumeric, 1-4 chars — to cover both numeric (`110`) and alpha (`DDN`) SAP storage type codes. Metric-row guard immediately after the regex match still prevents false positives on label rows.

## Inventory Completion goal-aware status pill

Follow-up to the [[Implement-LX25-Inventory-Completion]] feature shipped earlier today. The hero stat card's status pill used absolute thresholds (≥90% emerald / ≥70% amber / <70% red) and surfaced **AT RISK** on a 57.6% reading in May — which is actually **AHEAD** of the user's monthly accrual goal. User feedback:

> "We are currently in May, so by the end of May we should be at 50%. With us being at 57.6% early in May, we're ahead of the goal. Please adjust the at-risk to reflect the 10% per month goal."

### Business rule (now encoded as constants)

- `MONTHLY_GOAL_PCT = 10` — monthly accrual
- `ANNUAL_GOAL_PCT = 100` — yearly target by Dec 31
- YTD goal at end of month N = `min(N × 10, 100)` — clamps in December

### Threshold table (for May, where ytdGoal = 50%)

| Status | Comparison | Pill copy | Color |
|---|---|---|---|
| `ahead`     | `pct >= 50` | `AHEAD ↑ +X.Y pts`   | emerald |
| `on-track`  | `pct >= 40` | `ON TRACK` (no delta) | emerald |
| `at-risk`   | `pct >= 30` | `AT RISK ↓ -X.Y pts`  | amber   |
| `behind`    | `pct <  30` | `BEHIND ↓ -X.Y pts`   | red     |
| `unknown`   | `pct == null` | `NO DATA`            | muted   |

Thresholds clamp to 0 (so January's `ytdGoal - 20 = -10` becomes 0; any progress reads as on-track or ahead in early Jan when the count is just spinning up).

### Sample pill copy for May (50% goal)

| Pct | Status | Pill | Color |
|---|---|---|---|
| 57.6% | ahead    | `↑ AHEAD +7.6 pts`  | emerald |
| 50.0% | ahead    | `↑ AHEAD +0.0 pts`  | emerald |
| 45.0% | on-track | `ON TRACK`          | emerald |
| 40.0% | on-track | `ON TRACK`          | emerald |
| 39.9% | at-risk  | `↓ AT RISK -10.1 pts` | amber  |
| 35.0% | at-risk  | `↓ AT RISK -15.0 pts` | amber  |
| 30.0% | at-risk  | `↓ AT RISK -20.0 pts` | amber  |
| 29.9% | behind   | `↓ BEHIND -20.1 pts`  | red    |
| 20.0% | behind   | `↓ BEHIND -30.0 pts`  | red    |

Delta is intentionally suppressed on `on-track` because the value is between previous-month and current-month goals — a negative delta vs end-of-month would falsely read as failure. Delta is preserved on the returned shape for a future tooltip / detail view.

### Decisions / locked-in defaults

- **Static class maps over Tailwind safelist** — the JIT only sees class literals, so `text-${size}` would never ship. Listing utilities once in a `Record<FontSize, string>` keeps the JIT-visible classes co-located with the consuming code and avoids any safelist build-system coupling.
- **`display` font dropped** — not a real utility in this project; only `sans` / `serif` / `mono` are exposed in the picker.
- **Stacked-mode ignores `current_value`** — adding a sub-metric is the curator's deliberate opt-in to a different layout. Trying to render both would compete for the same visual real estate.
- **Trend + comparison only render in single-mode + only when 2+ history points exist** — stacked mode has its own visual weight; an arrow next to one of the sub-blocks would be confusing.
- **`lower_is_better` only flips the trend arrow color** — the spec mentioned an optional area-chart fill inversion; inspected and rejected (would muddy the dashed target line + above-target marker highlight which both rely on the accent).
- **Tabbed body at 4-section threshold** — Pattern note ([[Editable-Board-Dialogs]]) gained a "When to add tabs" decision rule pointing at this v12 work as the canonical example.
- **Sticky preview renders a real `<SqcdpCard>`, not just `<SqcdpChart>`** — v10 preview was chart-only; with v12's per-input typography + sub-metrics + prefix/suffix + decimals + polarity, a chart-only preview would have hidden most of the changes the curator was making.
- **`useWatch` once at form level** instead of 25+ individual `form.watch(...)` calls (the latter was the v10 pattern; doesn't scale).

The inline `<PrimaryValuePreview>` lives directly under the picker so they see the chosen size without flicking up to the sticky preview.
