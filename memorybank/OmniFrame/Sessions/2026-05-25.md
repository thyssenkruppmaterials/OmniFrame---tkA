# 2026-05-25

## RF Interface — full cinematic redesign

Redesigned the entire RF interface (home + 13 sub-form screens + bottom dock) around a shared `_shell/` module so every surface inherits the same cinematic shell: animated mesh backdrop, glassmorphic tiles with theme-aware accent halos, live-stat hero strip, glass-pill dock with sliding active indicator, and a unified stagger/page-push motion language. 100% theme-token (light/dark/custom auto-swap) with zero hardcoded color values.

### Built
- `src/features/rf-interface/_shell/` — new module: `MeshBackdrop`, `RFScreen`, `RFScreenHeader`, `RFCard`, `RFTile`, `RFHero`, `RFStatusPill`, `RFDock`, shared motion variants (`fadeUp`, `fadeUpFast`, `pagePush`, `staggerContainer`, `tapScale`).
- `src/index.css` — added 11 `--rf-accent-*` tokens (`:root` + `.dark` + `@theme inline`) plus 3 mesh-blob keyframe animations (paused under `prefers-reduced-motion`).
- `src/features/rf-interface/rf-interface.tsx` — replaced the inline `Dock` + `QuickActionButton` components, rebuilt the home view around `<RFHero>` + 12 `<RFTile>` instances in a 2-col grid, rewrote the `kitting-apps` sub-menu, restyled the `inventory`, `grs-cycle-count-old`, `grs-core-pulls`, `my-productivity`, `profile`, and `InboundScanForm` inline views with `<RFScreenHeader>`, swapped page-transition variants to `pagePush`, mounted `<MeshBackdrop>` once at the shell root.
- Sub-form sweep across 13 files in `src/components/ui/rf-*.tsx` — top-level back-button rows replaced with `<RFScreenHeader>`, kit-info / scan-mode subtitles preserved where present, status indicators routed to the `right={...}` slot.
