## Done — Kit Number on Kit Assembly Board cards
- Added the human-readable **kit number** to every card on the **Kit Assembly Board** (`KitKanbanBoard`), rendered as `Kit {kit_number}` under the title (e.g. `Kit 424 Inlet #1`). The user's "Kit 10 - Inlet" example maps directly to `kit_number` — that field already embeds the descriptor ("Inlet", "Rotor", etc.).
- **Key gotcha:** `kit_number` is **not** a column on `kit_kanban_tasks` (verified via Supabase MCP — the live table only has serial / PO / build number / progress). It lives on `RR_Kitting_DATA`. So the service **enriches** kanban tasks from `RR_Kitting_DATA`, keyed by `kit_serial_number` per [[Kit-Serial-Scoping]] (no PO fallback — multi-kit POs).
- **Files:** `kit-kanban.service.ts` (`getTasksByColumn` batch-enrich into `Map<serial, kit_number>` + new `getKitNumberBySerial`), `kit-kanban-board.tsx` (card line + `Boxes` icon + memo comparator + realtime preserve-on-UPDATE / patch-on-INSERT, since the realtime payload can't carry `kit_number`).
