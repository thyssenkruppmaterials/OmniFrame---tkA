# Session 2026-05-05 — Outbound stat cards converted to clickable filter pills

## Goal

User asked: *"In the outbound apps, in both the delivery status page and data manager page, please update the stat cards to be clickable filters like how it is setup in the inventory apps page, in the inventory counts tab."*

Mirror the [[Stat-Card-Clickable-Filter-Pills]] affordance from `manual-counts-search.tsx` onto the two Outbound Apps surfaces so each numeric pill becomes a one-click filter for the table beneath it.

## What shipped

### Delivery Status Manager (`src/components/delivery-status-manager.tsx`)

- Added `cardFilter` state — a single discriminated union covering `shippingPoint | daysOpen | tka` so the four cards (Total Deliveries, Days Open, TKA Non-Controllable, Deliveries PGI) share one source of truth.
- Wired filter into the existing `sortedAndFilteredData` `useMemo` as a final pass; reset to page 1 via dedicated `useEffect` whenever `cardFilter` changes; cleared by `handleClearFilters`.
- Special handling for the **TKA Non-Controllable** card: LiftFan / WAWF rows are normally excluded by the upstream `openOnly` Supabase query, so the `useDeliveryStatus({ openOnly })` argument is now `showOpenOnly && !showJS01Only && !showDeletedOnly && cardFilter?.type !== 'tka'`. Toggling a TKA pill triggers a re-fetch with the broader dataset.
- Restyled all four cards with the inventory-counts chrome — opacity colour palette, `Click to filter` ↔ `Filtered · clear` chip, `aria-pressed` + `ring-2 ring-{color}-500/60` on active pills, `tabular-nums` on counts.
- The **Deliveries PGI** card keeps its calendar popover (the date IS the filter) but uses the new card chrome and an info-tile-style value display for visual consistency.
- Removed unused `Separator` and `Users` imports; added `Clock` for the Days Open icon.

### Outbound Data Manager (`src/components/outbound-data-manager.tsx`)

- Added `showPendingOnly` filter state so the Pending pill in the Delivery Status card can act as a clickable filter; threaded it through `fetchStatusFilteredData` (calls `service.fetchByStatuses(['pending'], cutoffDate)`) and `filteredData`.
- Introduced `StatusFilterKey` + `activeStatusFilter` derived state + `setStatusFilter(key | null)` + `toggleStatusFilter(key)` helpers — single point that flips the five mutually-exclusive booleans (`showCriticalOnly`, `showPendingOnly`, `showWavedOnly`, `showPickedOnly`, `showShippedOnly`) and resets pagination.
- Refactored the legacy "More" dropdown menu items to call `toggleStatusFilter`, eliminating duplicated boilerplate; added a new **Pending Only** entry; refactored the in-page Clear Filter button to call `setStatusFilter(null)`.
- Restyled all four cards with the inventory-counts chrome. **Today** pills (Waved Today, Picked Today, Packed Today, Final Packed Today) are date-scoped — they don't map to a status filter — so they render as static `infoTileBase` tiles, mirroring the Variance summary tile inside the Inventory Counts tab.
- Removed unused `Separator` import.
