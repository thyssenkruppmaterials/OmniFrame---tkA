## Purpose / Context

Phase E of [[Plan-Multi-Session-Agent-Master]] delivers a resumable six-step CustomTkinter wizard that blocks the Agent Master main window until `master_config.yaml` exists and every configured worker has a restricted-ACL service key. It automates the operator runbook in [[Implement-Phase10-Service-Key-First-Rollout]] without touching `omni_agent/agent.py` or the web SPA.

### Exit criteria mapping

1. Fresh box detection via missing `master_config.yaml` or any `agents/<id>/agent_service_key.txt`.
2. Six gated steps; `[Next]` / `[Finish]` driven by panel `is_valid()`.
3. SAP probe typed errors + session-count warning when `len(sessions) < master.workers`.
4. Static pairing defaults (W6 `auto_start=false` when workersâ‰Ą6); duplicate `(conn,sess)` rejected.
5. Register step reuses `write_service_key()` + `build_reregister_url()` from Phase D.
6. SAPLogon autodetect + skip checkbox.
7. Confirm writes config, clears `wizard_state.json`.
8. Resumable `wizard_state.json` until Finish.
9. Keys at `~/.omniframe/agents/<worker-id>/agent_service_key.txt` with Phase D ACL helper.
10. No Phase F/G scope (orphan adoption, PyInstaller, Settings persistence).
